SpringBoot - SpringApplication

2019-04-30


Application

기존에 만들어지는 방법으로 실행시키면 스프링부트가 제공하는 기능을 전부 사용하기 어렵습니다.
참고 : 스프링공식문서

그래서 위와 같은 코드로 app 인스턴스를 생성하여 실행시키는 방법을 사용합니다.


Debug 사용하기

디버그 모드는 자동설정되는 사항들이 적용되는지 여부를 알려줍니다.
Edit Configuration을 클릭합니다.

  • VM options -Ddebug
  • Program arguments --debug

이 두가지를 사용하여 디버그 모드를 활성화 시킬 수 있습니다.
두 가지 중 하나만 선언하시면 구동됩니다.

디버그 모드가 활용된 모습입니다.


배너 사용하기

배너는 스프링부트 실행시 나오는 배너(문구)입니다.
이 것이 배너!!


파일로 배너 만들기

main - resources 경로에 banner.txt를 생성하고 원하는 문구를 넣으면 됩니다.

문구를 넣고 실행한 모습입니다.

파일을 타경로에 생성하였다면 application.properties에서 classpath를 통해 경로를 지정해주면 됩니다.


배너 기능 사용하기

추가적으로 버전도 나타낼 수 있습니다.
기타 여러가지 기능이 있는데 MANIFEST.MF가 있어야 가능한 기능이 있습니다.(application 버전 등..)
이는 packaging을 통하여 jar파일을 생성해 구동하여야 합니다.

packaging…
jar를 실행하니 application 버전이 출력됩니다.


코드로 배너 만들기

코드로 배너를 만들 수 있습니다.

생성한 인스턴스를 사용하여 setBanner함수를 사용하여 출력하고싶은 문구를 출력할 수도 있습니다.

지금까지 생성한 배너를 출력하고 싶지 않다면, setBannerMode를 OFF모드로 변경하여 미출력하게 할 수 있습니다.
